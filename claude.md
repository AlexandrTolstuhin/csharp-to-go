# Инструкции для Claude по работе с курсом

## Общие принципы

- Все материалы на **русском языке**, комментарии в коде — тоже
- Commit messages на русском
- Ориентация на **C# разработчиков**: постоянное сравнение C# ↔ Go
- Акцент на **производительность, GC и идиоматичный Go**
- **Практические примеры** с реальным кодом
- Высокий приоритет: идиоматичность → практичность → сравнение с C# → влияние на GC

## Git workflow

После завершения каждого файла или логической части — **коммит и push**.

```bash
git add <файлы>
git commit -m "Добавлен раздел 2.1: Горутины и каналы"
git push origin master
```

**Формат commit message**:
```
[Раздел/секция] Краткое описание

Добавлено:
- что добавлено

Обновлено:
- что изменено
```

❌ **Не коммитить**: незавершённые файлы (кроме WIP), заглушки, временные файлы.

## Автоматизация

Четыре скрипта поддерживают проект в актуальном состоянии. Запускать **в этом порядке** при добавлении нового файла:

| # | Скрипт | Что обновляет | Триггер |
|---|--------|---------------|---------|
| 1 | `npx doctoc <файл>.md --github --notitle` | `## Содержание` в файле | изменились заголовки `##`/`###` |
| 2 | `python update_readme.py <part>` | `<!-- AUTO: MATERIALS -->` в `part*/README.md` | новый файл или изменились `##`-заголовки |
| 3 | `python update_root_readme.py` | `<!-- AUTO: STRUCTURE -->` в корневом `README.md` | после шага 2 |
| 4 | `python update_nav.py` | `<!-- AUTO: NAV -->` во всех файлах | новый файл или изменился порядок файлов |
| 5 | `python update_context.py` | `.context-summary.md` (структура, прогресс) | изменился состав или размер файлов |

Для проверки без записи: `--dry` флаг у `update_readme.py`, `update_root_readme.py`, `update_nav.py`.

### Правила AUTO-блоков

⛔ **ЗАПРЕЩЕНО редактировать вручную** содержимое между любыми `<!-- AUTO: ... -->` / `<!-- /AUTO: ... -->` маркерами — оно перезапишется скриптом:
- `<!-- AUTO: MATERIALS -->` в `part*/README.md`
- `<!-- AUTO: STRUCTURE -->` в корневом `README.md`
- `<!-- AUTO: NAV -->` во всех файлах
- `<!-- AUTO: STRUCTURE -->`, `<!-- AUTO: PROGRESS -->`, `<!-- AUTO: VERSION -->` в `.context-summary.md`

### Что редактировать вручную

| Файл | Редактируемые секции |
|------|---------------------|
| `README.md` (корень) | Прогресс-бар badge, `## О курсе`, `## Полезные ресурсы` |
| `part*/README.md` | `## Описание` |
| `.context-summary.md` | `### Репозиторий`, `### Особенности курса`, `### Технические детали` |

⛔ **ЗАПРЕЩЕНО** добавлять новые разделы в `.context-summary.md` без явной просьбы пользователя.

### Версия проекта

```bash
python update_context.py --version "0.9.0 (краткое описание изменений)"
```

Semver логика: `PATCH` — исправления/дополнения, `MINOR` — новый раздел или завершена часть, `MAJOR` — курс полностью завершён. Текущую версию читать из `.context-summary.md` между `<!-- AUTO: VERSION -->` маркерами.

## Процесс написания статей

**Писать инкрементально** — не генерировать весь файл целиком за один раз:

1. Создать файл с заголовком и `## Содержание`
2. Писать по одному подразделу → сохранять
3. После завершения: doctoc → update_readme → update_root_readme → update_nav
4. Финальный коммит

**Размер**: 500–1500 строк за итерацию, максимум 2000.

## Особенности контента

### Акценты на идиоматичности

Показывать не только «как сделать», но и «как принято»:

```go
// ❌ Не идиоматично
func GetUser(id int) (*User, error) { ... }

// ✅ Идиоматично
func getUser(id int) (*User, error) { ... }
```

Типичные ошибки C# разработчиков: попытка использовать наследование, игнорирование ошибок, неправильная работа с nil, излишнее использование интерфейсов, getter/setter для всего.

Паттерны миграции: LINQ → циклы, async/await → горутины, DI → композиция, exceptions → errors.

### Актуализация материалов

Дополнять существующие разделы при:
- Новой версии Go с изменениями в обсуждаемой теме
- Обнаруженных пробелах или неточностях
- Связанных темах, выявленных при работе над новым разделом

После дополнения — отдельный коммит с описанием изменений.

### Производительность

Включать: benchmarks с цифрами, memory allocations, GC pressure, сравнение с C#, примеры pprof.

### Практические проекты

Каждый проект: реальная задача + поэтапная реализация + полное решение в `<details>` + дополнительные задания + тестирование.
